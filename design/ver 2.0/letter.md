About the Context MCP server: how does the agent know about Blazor features?
Some of it may be in the LLM, but some features are recent. The agent can look on the internet and in Microsoft's documentation, but it needs more tokens to find and understand the information. The agent can use a Context MCP server registered in a client like VS Code, Cursor, or the CLI. The MCP server exposes tools with descriptions that clients can discover and invoke, and the agent knows what to call. It can also use prompts to guide how and when to use the MCP server.

I have used the MCP server at https://context7.com for free for the game project. It has recent information, but it is public. How can I code using internal company docs, for example, by using a Smiley NuGet package like STI.Utilities or a NuGet package with OpenAPI clients? This is what dev-context-mcp-server tries to resolve.

Each NuGet package is deployed with a readme.md file and XML comments in the nupkg file. This info is used as context for the agent when the prompt mentions internal NuGet packages.
