# Dashboard UI

`ui/` is a local Next.js (App Router, TypeScript) dashboard that visualizes MCP
server analytics and the current index inventory. Its **server-side route
handlers proxy** to the .NET server's [HTTP API](/http-api.md), so the browser
only ever talks to the dashboard's own origin — never directly to the .NET host,
so no CORS configuration is required. It implements Stage 3 of `design/ver 2.0`.

# Schema

## Pages

| Page | File | Shows |
|------|------|-------|
| Analytics | `ui/app/page.tsx` | KPIs (total calls, success rate, avg & p95 latency), calls-over-time area chart, status distribution, per-tool breakdown, latency (avg/p50/p95/max), recent calls. A date-range and bucket control drives every panel. |
| Indexed Context | `ui/app/context/page.tsx` | Read-only inventory: environments, NuGet count, per-environment breakdown (`NuGetInventoryTable`), and the last indexing run (`LastRunTable`). |

## Proxy route handlers

| Route | Proxies to |
|-------|-----------|
| `ui/app/api/analytics/[endpoint]/route.ts` | `/api/analytics/*` (allow-list: `summary`, `timeseries`, `tools`, `users`, `tool-results`, `recent`); `404` for unknown endpoint, `502` if upstream unreachable. |
| `ui/app/api/analytics/recent/[id]/route.ts` | `/api/analytics/recent/{id}` |
| `ui/app/api/context/route.ts` | `/api/context` |
| `ui/app/api/context/last-run/route.ts` | `/api/context/last-run` |

Proxy target: `MCP_ANALYTICS_API_BASE_URL` (default `http://127.0.0.1:2222`),
overridable via `ui/.env.local`.

## Typed client (generated)

- `ui/openapi.json` — OpenAPI document exported from the .NET server.
- `ui/lib/generated/schema.d.ts` — types generated by `openapi-typescript` (do **not** hand-edit).
- `ui/lib/client.ts` — an `openapi-fetch` client (`apiClient`) with a relative `baseUrl` so calls route through the proxy.

Regenerate after the server contract changes:

```bash
curl http://127.0.0.1:2222/openapi/v1.json -o ui/openapi.json
npm run gen:api
```

# Examples

```bash
# Prerequisite: the .NET server running with analytics enabled (default :2222).
cd ui
npm install
npm run dev      # http://localhost:3000
```

Data flow:

```text
browser ──► /api/analytics/* (Next.js route handler)
                     │  server-to-server fetch
                     ▼
        MCP server  /api/analytics/{summary,timeseries,tools,recent}
                     │
                     ▼
                 analytics.db (SQLite)
```

Send an `X-User-Name` header when generating traffic so the user audit populates.
See [HTTP API](/http-api.md) and [Analytics](/analytics.md).
